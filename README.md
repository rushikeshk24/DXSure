## 🏛️ System Architecture 
DXSure operates on a decoupled **MERN** infrastructure, separating pure operational logic from high-fidelity UX representations.

*   **Frontend (Canvas):** React (Vite) utilizing Tailwind CSS v4.
*   **Backend (Core):** Node.js / Express.
*   **Database (Store):** MongoDB (Mongoose Schema mapping).
*   **Aesthetic Sync:** `StitchMCP` (Project ID: `14209085376760565069`) - The UI leverages a strict **"Dark-Emerald"** visual hierarchy utilizing `neutral-950` backdrops and `emerald-600` accents.

## 🚀 Features

### Admin Privileges
- **Track Record:** Full audit trail rendering user logins and systemic changes.
- **Ticketing & DayPlan Management:** Real-time visibility into internal IT escalations and employee task cycles.
- **Financial Day Book (P&L):** Encrypted routing for high-level vendor and petty-cash aggregations.

### Employee Dashboard
- **Client Pipelines:** Enquiries, Leads, Follow-ups, and native Registrations.
- **Vendor Tracking:** Procurement registration alongside Pettycash logging.
- **Interactive DayPlans:** Submission engines for granular daily work logs.

## 🛠️ Environment Initialization 

The `client` and `server` modules require isolated boots logic perfectly mapped via CORS `(localhost:5173 ↔ localhost:5000)`.

### 1. Booting The API (Server)
```bash
cd server
npm install 
# Ensure .env exists with MONGO_URI and JWT_SECRET
npm start 
```

### 2. Booting The UI (Client)
```bash
cd client
npm install
npm run dev
```

## 🎨 Design System: The Digital Architect
All UI refactors abide strictly by our *Architectural Intelligence* token sets.
1. **No-Line Rule:** Borders are substituted with ambient container nesting (`neutral-900` over `neutral-950`).
2. **Glassmorphism:** Navigation and modals leverage backdrop filters (`backdrop-blur-md`) rather than heavy opacities.
3. **Data Cues:** Dynamic loading states simulate real-world backend API fetches across inputs and tables dynamically generated by our MCP synchronization layer.
